一、Gurobi JLL 版本冲突修复
--------------------------
问题：Gurobi.jl 默认走 JLL artifact 拉取 v13 二进制，忽略本地 v12.0.3 安装。
即使 GRB_LICENSE_FILE 指向有效许可，许可版本与 JLL 版本不匹配即拒绝创建 Env。
修复：GUROBI_JL_USE_GUROBI_JLL=false + GUROBI_HOME + Pkg.build("Gurobi")，
三变量需同时设置。已写入 CLAUDE.md 防止后续会话重犯。

二、Julia 模块导出
------------------
五个模块全部缺 export 语句，using .Module 只能引入模块名无法引入函数名，
导致 UndefVarError。补完后项目一遍跑通，16 个 ε 点 × 3 条 frontier 共计
48 次 Gurobi 调用均求解成功。代码层面无其他 bug。

三、GitHub 推送链路
-------------------
环境：MacBook M4 + 国内网络。
路径：brew install gh → git init/commit → gh auth login → gh repo create → push。
踩坑清单：
  (1) gh auth web 浏览器模式被墙，换 paste token 方式。
  (2) token 缺 read:org scope，编辑补上。
  (3) HTTPS push HTTP/2 握手失败，换 git config http.version HTTP/1.1 仍 empty reply。
  (4) 换 SSH，已有一对 ed25519 key，公钥上传 GitHub → 推送成功。
最终：https://github.com/WLiu1949/cvar-portfolio-dro
